# ADR-0000: Architecture Decision Record Template

This is a template file for documenting architectural decisions. It explains the structure and requirements for recording non-trivial, hard-to-reverse trade-offs.

## Context

Describe the background, context, and state of the system before this decision. Why was a decision necessary? What problem are we solving?

## Decision

* **Status:** `proposed | accepted | deprecated | superseded by ADR-NNNN`
* **Date:** YYYY-MM-DD

What is the exact decision we reached? State it in clear, active voice (e.g., "We will use manual SQL with SQLite instead of a heavy ORM").

## Consequences

* **Positive:** What are the positive outcomes of this decision?
* **Negative:** What are the negative outcomes, trade-offs, or downstream complications that we must accept?
